Comment utiliser l’interface
1. Créez d’abord une brique de contenu.
2. Ajoutez ensuite une version de cette brique.
3. Créez un type de déclencheur, puis un scénario.
4. Ajoutez une ou plusieurs étapes au scénario.
5. Créez enfin un événement pour tester votre logique.

Créer une brique de contenu

Une brique est un morceau de message réutilisable : alerte, conseil, rappel, conclusion…
Nom unique utilisé par le système.
Exemple : ne montrer ce contenu qu’une fois par événement, avec un délai de répétition de 7 jours.

Briques existantes

Créer une version de contenu

Une même brique peut avoir plusieurs versions : brouillon, publiée, archivée…
Vous pouvez utiliser des variables comme {lieu}, {source_url}, {territory}…

Créer un type de déclencheur

Un type de déclencheur correspond à une famille d’événements : radar, orage, changement de vitesse…

Types existants

Créer ou modifier un scénario

Un scénario correspond à une situation métier : orage, radar, changement de limitation, alerte transport, message manuel… Les messages précis se règlent ensuite dans la page Étapes des scénarios.
Plus la priorité est élevée, plus ce scénario sera traité comme important en cas de concurrence avec d’autres messages.

Scénarios existants

Cliquez sur “Gérer les étapes” pour voir et modifier toute la séquence d’un scénario au même endroit.

Vue d’ensemble des étapes d’un scénario

Ici, vous modifiez toute la séquence en une seule fois. L’objectif est de comprendre immédiatement : si l’événement est prévu à telle date, quand chaque message partira.

1. Choisir le scénario à régler

Exemple : si vous indiquez vendredi 18h et qu’une étape part 48h avant le début prévu, l’aperçu affichera mercredi 18h.

2. Aperçu lisible de la séquence

Les anciennes notions “heures minimales / heures maximales” sont masquées. Vous renseignez simplement combien d’heures avant la date de référence le message doit partir.

Créer un événement

Un événement est un cas concret : un orage annoncé, un radar installé, un changement de limitation…
Vous pouvez laisser ce champ vide. Dans ce cas, l’événement sera créé avec un contexte vide et la date actuelle sera utilisée pour les dates non renseignées.
